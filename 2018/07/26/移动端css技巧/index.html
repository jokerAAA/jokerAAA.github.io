移动端css技巧

一些小技巧（持续更新中）

  • 单行文字超出显示省略号：text-overflow:ecllipse;overflow:hidden;white-space:nowrap;

  • 两行文字超出显示省略号：如果需要居中，添加一层标签即可。

  • rem单位一些问题

    • 在有边框的元素中，使用rem可能在部分机型上无法显示，使用px做边框单位同时使用lineheight做垂直居中时可能不准
    • 一般不推荐使用rem做字体单位，但是使用rem做元素单位，使用px作字体单位时，在一些小屏如iphone5上可能出现换行的现象
    • rem使用媒体查询的方式来实现时，可能会漏掉部分非常规屏幕的手机

  • label元素在ios和安卓上表现不一致，存在事件是否冒泡的区别，具体使用时注意即可；

  • 局部滚动和iscroll

    • 在ios上使用局部滚动会卡顿，添加-webkit-overflow-scrolling属性，会有额外bug
    • iscroll5滚动流畅，偶尔出现底部无法上拉情况。
    • 一般不推荐局部滚动，在hybridApp中，使用局部滚动时，native的下拉刷新会受到影响，除某些特殊页面设计外，普通页面推荐尽量拉深页面

  • 遮罩层滚动：在移动端使用弹窗时会出现遮罩层跟随滚动的情况

    • 在mask上阻止touchmove事件的冒泡，常用方案，但是一直未生效

  • ios上readonly会出现光标，在input上添加 unselectable=”on” onfocus=”this.blur()” 即可；安卓表现正常;

  • ios上div等标签注册事件会失效，添加cursor:pointer可以生效

  • 关闭自动填充：autocomplete=off，直接添加在input上即可

  • 启用定时器如果出现误差，可以先计算与服务器上的时间差，注意本地/测试/线上服务器的时间差和时区设置；ios如果定时器在后台不运行，可以通过worker来解决，worker会单独开启线程；

  • z-index会受到transform和opacity的影响，本质是transform会生成合成层，位于普通层级之上，解决方案 是取消transform属性，或者设置transformZ属性；

  • contenteditable:html5新增属性，兼容ie，给元素添加后元素变为可编辑，取代textarea，其中的内容可以添加各种样式；

webkit特性

  • -webkit-appearance：none；取消默认input边框，结合border-radius：0来使用

  • element::-webkit-scroll-bar：{display：none}；取消滚动条

  • -webkit-overflow-scrolling : touch;

    在ios上使用局部滚动时，会出现卡顿的现象，添加该属性会让局部滚动变流畅，也会带来别的bug，目前已知的有屏幕可拖动、input偶尔异常等

  • -webkit-tap-highlight-color:点击高亮色，设置为rgba色，透明度为0时消失
